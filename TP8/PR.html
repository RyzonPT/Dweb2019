Abstract:

Este trabalho consistiu em criar uma app WEB estruturada com duas routes principais, API e Interface.

Para isso, foi gerado um novo projeto com a ajuda das funcionalidades do node Express, npm mongoose e Axios.

Diferente das aplicações web anteriores, esta recebe todos os pedidos do cliente para a Route da interface a qual pede, posteriormente, os dados à Route da API que irá depois buscar os dados à camada de persistência.

De facto, esta abordagem permo-te um maior controlo pelo que o cliente nunca tem contaco direto com a camada de dados o que se traduz em maior segurança.

Por outro lado, gerir a aplicação em 3 camadas distintas, interface, funcionalidade e persistência de dados, faz com que toda a aplicação esteja bem mais organizada.

Podemos ainda ver que este tipo de arquitetura do sistema nos permite separar estas camadas em diferentes servidores, o que pode ser uma vantágem na gestão dos servidores, quer seja por performance ou segurança.
